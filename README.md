## üö® Critical Success Factors

### Essential Patterns
Always remember these two critical patterns:

**Permission Management:**
```solidity
// ‚úÖ ALWAYS call both permissions
FHE.allowThis(encryptedValue);        // Contract permission
FHE.allow(encryptedValue, msg.sender); // User permission
```

**Encryption Binding:**
```typescript
// ‚úÖ Encryption user MUST match execution signer
const input = await createEncryptedInput(addr, userAddress)...;
await contract.connect(signer).operation(...);  // Same signer!
```

### Common Mistakes to Avoid
- ‚ùå Forgetting FHE.allowThis() ‚Üí Contract cannot use value
- ‚ùå Signer mismatch ‚Üí "decryption failed" error
- ‚ùå View functions with encrypted values ‚Üí Not allowed
- ‚ùå Invalid input proofs ‚Üí Decryption fails
